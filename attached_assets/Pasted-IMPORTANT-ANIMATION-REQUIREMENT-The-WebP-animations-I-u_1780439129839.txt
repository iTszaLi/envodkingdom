IMPORTANT ANIMATION REQUIREMENT

The WebP animations I uploaded must NOT autoplay independently.

I want Apple-style scroll-controlled storytelling where the animation progress is directly linked to the user's scroll position.

Behavior:

- When the user scrolls down, the animation moves forward.
- When the user scrolls up, the animation moves backward.
- If the user stops scrolling, the animation stops exactly on that frame.
- Animation progress must be synchronized 1:1 with scroll progress.
- Use GSAP ScrollTrigger with scrub enabled.
- Pin sections during animation playback.
- Smooth interpolation between frames.
- No automatic looping.
- No autoplay.

Implementation Requirements:

- Create cinematic scroll sections.
- Use Canvas rendering for best performance.
- Preload WebP frames.
- Lazy load assets.
- Support desktop, tablet, and mobile.
- Maintain 60fps where possible.
- Use Lenis smooth scrolling integrated with ScrollTrigger.
